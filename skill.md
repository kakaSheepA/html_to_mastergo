# 一、Color Design Skill — 颜色系统

## 概述

本组件库的颜色系统由三大类构成：**主色（Primary Colors）​**、**功能色（Functional Colors）​** 和 **中性色（Neutral Colors）​**。每种颜色均以 0～100 的色阶步长进行定义，数值越大颜色越深，便于在不同场景下统一调用。所有色值均提供精确的 HEX 值，确保跨平台、跨设计工具的一致性。

---

## 1.3 中性色 Neutral Colors

中性色基于黑色 `#000000` 的透明度叠加生成，用于文字、线条、背景三类场景，建立页面的视觉层级与阅读秩序。

| 色阶范围 | 典型用途 |
|---------|---------|
| 0～20 | 浅色背景、标签底色、Hover 背景 |
| 30～50 | 图标填充、按钮默认色、基准品牌色 |
| 60 | Hover / Active 加深态 |
| 70～100 | 深色文字、强调状态、深色模式 |

# 二、Typography Design Skill — 文字系统

## 概述

文字系统定义了组件库中字体、字号、行高的使用规范，旨在建立清晰的信息层级，保障页面内容的可读性与一致性。针对 EE 侧业务页面内容多且复杂的特性，系统以 **14/20** 和 **12/18**（字号/行高，单位 px）作为基础配置。

---

## 2.1 字体 Typeface

### 西文字体 — Inter

本系统西文内容统一使用 **Inter** 字体，该字体专为屏幕阅读优化，具备良好的字重覆盖和多语言支持。

- 官方网站：<https://rsms.me/inter/>
- GitHub 仓库：<https://github.com/rsms/inter>

### 中文字体 — FD 字体

中文内容使用内部定制的 **FD 字体**，字体文件请联系设计人员单独获取，不对外公开分发。

---

## 2.2 基础字号与行高 Font Size & Line Height

| 字号 | 行高 | 适用场景 |
|------|------|---------|
| 14px | 20px | 标准正文、主要说明文字 |
| 12px | 18px | 辅助说明、次要信息、标注文字 |

> **使用原则：​** 一个页面中建议只存在 **3～4 种字体大小**，避免字号过多导致视觉层级混乱。EE 侧业务优先以 14px 为主体字号，12px 用于补充说明。

---

## 2.3 段落规范 Paragraph

段落文字在超出 2 行时需提供截断处理（如 ellipsis 或展开交互），避免过长的文本块影响页面扫读效率。

**12px / Line-height 18px**
> 自9月11日起，乘客端及司机端的开屏和弹窗删掉推送信息后就规范内容进行了更新，请前往【集团端内应用规范】和【网约车端内应用规范】页面下载使用。

**14px / Line-height 20px**
> 自9月11日起，乘客端及司机端的开屏和弹窗删掉推送信息后就规范内容进行了更新，请前往【集团端内应用规范】和【网约车端内应用规范】页面下载使用。

---

## 2.4 使用规则总结

| 规则 | 说明 |
|------|------|
| 字号种类控制 | 单页面内字号种类不超过 3～4 种 |
| 行高配套 | 字号与行高须成对使用，不可单独调整 |
| 西文字体 | 统一使用 Inter，禁止混用其他西文字体 |
| 中文字体 | 使用 FD 字体，需向设计人员获取字体文件 |
| 段落截断 | 超出 2 行的段落内容须提供截断或展开处理 |

# 三、Border Radius Design Skill — 圆角系统

## 概述

圆角系统定义了组件库中不同类型 UI 元素所使用的圆角半径规范，通过统一的圆角梯度建立视觉一致性。不同组件类型对应不同的圆角值，形成从方正到圆润的视觉层次。

---

## 3.1 圆角梯度 Border Radius Scale

| 级别 | 圆角值 | 适用组件类型 |
|------|--------|------------|
| R2 | 2px | 小标签（Tag）、内部卡片辅助元素 |
| R4 | 4px | 按钮（Button）、表单输入框（Form） |
| R6 | 6px | 图片容器、内部卡片（Inner Card） |
| R8 | 8px | 卡片 / 信息承载容器（Card / Container）**​【P0 优先级】​** |

---

## 3.2 各组件圆角规范

### 卡片 / 信息承载容器 Card / Container
使用 **8px** 圆角，为 P0 级别的核心容器组件，是页面中最主要的内容承载单元。

### 按钮 / 表单 Button / Form
使用 **4px** 圆角，保持适度的方正感，兼顾操作控件的功能性与视觉亲和力。

### 小标签 Tag
使用 **2px** 圆角，接近方形，适合密集排列的状态标签、分类标签等小型元素。

### 图片 / 内部卡片 Image / Inner Card
使用 **6px** 圆角，适用于卡片内嵌的图片区域或子卡片模块。

---

## 3.3 使用规则总结

| 规则 | 说明 |
|------|------|
| 层级对应 | 圆角值随组件层级升高而增大，容器 > 图片 > 按钮 > 标签 |
| 禁止随意调整 | 不同组件类型的圆角值已固定，不可跨类型混用 |
| P0 优先保障 | 卡片容器的 8px 圆角为 P0 级规范，必须严格执行 |
| 一致性原则 | 同类组件在同一页面内圆角值须保持统一 |

# 四、Spacing Design Skill — 间距系统

## 概述

间距系统通过统一的数值规范，定义组件库中元素与元素之间的纵向、横向距离关系，以及分组边界的视觉处理方式。合理的间距层次能够帮助用户清晰感知信息的主次与归属关系。

---

## 4.1 间距基准规则

所有间距值须满足 **4 的倍数**，最小间距为 **4px**。

---

## 4.2 间距级别 Spacing Scale

| Token | 值 | 语义 | 典型用途 |
|-------|----|------|---------|
| Mini | 4px | 极小间距 | 图标与文字之间、标签内边距、紧密排列元素 |
| Small | 8px | 小号间距 | 同组内相邻元素间距、按钮内部元素间距 |
| Middle | 16px | 中号间距 | 组件内部分区间距、表单项之间的间距 |
| Large | 24px | 大号间距 | 不同信息组之间的分隔、卡片内模块间距 |

---

## 4.3 纵向间距关系 Vertical Spacing

- 同组内相邻元素：**4px / 8px**
- 组件内不同区块：**12px / 16px**
- 独立信息组之间：**24px**

---

## 4.4 横向间距关系 Horizontal Spacing

横向排布的元素应优先使用**栅格布局（Grid Layout）​** 进行定位，间距值仍遵循 4px 倍数原则，不可使用任意值。

---

## 4.5 分组关系 Grouping

当多个元素需要被归为同一信息组时，除使用间距区分外，还可结合以下视觉手段强化分组边界：

- **背景色差异**：使用不同层级的中性背景色区分分组
- **分隔线**：使用 Grey 6 `#E0E0E0` 或 Grey 7 `#F0F0F0` 的细线进行区域分割

---

## 4.6 使用规则总结

| 规则 | 说明 |
|------|------|
| 4px 倍数原则 | 所有间距值必须为 4 的倍数，最小 4px |
| 禁止任意值 | 不可使用 3px、5px、7px 等非规范值 |
| 层级对应 | 间距越大，信息组之间的独立性越强 |
| 横向使用栅格 | 横向排布须配合栅格系统 |
| 分组辅助手段 | 可结合背景色或分隔线强化分组，不可仅依赖间距 |
| 页面间距种类 | 单页面内建议间距级别不超过 4 种 |

# 五、Shadow Design Skill — 阴影系统

## 概述

阴影系统共定义 **4 个层级**，从高到低对应不同的视觉海拔感（Elevation）。每个层级由 3 层叠加阴影构成，层级越高，阴影越扩散、越明显。所有阴影颜色均使用黑色半透明叠加。

---

## 5.1 阴影级别 Shadow Scale

### Shadow Level 1 — 最高层级（High）

适用于全局弹窗（Modal）、抽屉（Drawer）、全屏覆盖浮层等最高层级组件。

| 层 | X | Y | 模糊（Blur） | 扩展（Spread） |
|----|---|---|------------|--------------|
| 层 1 | 0 | 4 | 14 | 0 |
| 层 2 | 0 | 12 | 16 | 16 |
| 层 3 | 0 | 10 | 16 | 0 |

```css
box-shadow:
  0px 4px 14px 0px rgba(0, 0, 0, α),
  0px 12px 16px 16px rgba(0, 0, 0, α),
  0px 10px 16px 0px rgba(0, 0, 0, α);
```

### Shadow Level 2 — 次高层级（Medium-High）

适用于下拉菜单（Dropdown）、气泡提示（Popover）、右键菜单等次级浮层组件。

| 层 | X | Y | 模糊（Blur） | 扩展（Spread） |
|----|---|---|------------|--------------|
| 层 1 | 0 | 0 | 8 | 0 |
| 层 2 | 0 | 6 | 12 | 12 |
| 层 3 | 0 | 6 | 10 | 0 |

```css
box-shadow:
  0px 0px 8px 0px rgba(0, 0, 0, α),
  0px 6px 12px 12px rgba(0, 0, 0, α),
  0px 6px 10px 0px rgba(0, 0, 0, α);
```

### Shadow Level 3 — 次低层级（Medium-Low）

适用于卡片悬浮态（Card Hover）、工具栏（Toolbar）、固定头部等需要轻微抬升感的组件。

| 层 | X | Y | 模糊（Blur） | 扩展（Spread） |
|----|---|---|------------|--------------|
| 层 1 | 0 | -2 | 4 | 0 |
| 层 2 | 0 | 2 | 6 | 6 |
| 层 3 | 0 | 2 | 6 | 0 |

```css
box-shadow:
  0px -2px 4px 0px rgba(0, 0, 0, α),
  0px 2px 6px 6px rgba(0, 0, 0, α),
  0px 2px 6px 0px rgba(0, 0, 0, α);
```

### Shadow Level 4 — 最低层级（Low）

适用于普通卡片（Card）默认态、列表项、输入框聚焦等需要最轻微层次感的组件。

| 层 | X | Y | 模糊（Blur） | 扩展（Spread） |
|----|---|---|------------|--------------|
| 层 1 | 0 | 2 | 4 | 0 |
| 层 2 | 0 | 3 | 6 | 3 |
| 层 3 | 0 | 2 | 6 | 0 |

```css
box-shadow:
  0px 2px 4px 0px rgba(0, 0, 0, α),
  0px 3px 6px 3px rgba(0, 0, 0, α),
  0px 2px 6px 0px rgba(0, 0, 0, α);
```

---

## 5.2 层级与场景对应

| 层级 | 适用场景 |
|------|---------|
| Level 1（High） | 全局弹窗、抽屉、全屏覆盖浮层 |
| Level 2（Medium-High） | 下拉菜单、气泡提示、右键菜单 |
| Level 3（Medium-Low） | 卡片悬浮态、工具栏、固定头部 |
| Level 4（Low） | 普通卡片默认态、列表项、输入框聚焦 |

---

## 5.3 使用规则总结

| 规则 | 说明 |
|------|------|
| 层级唯一性 | 同一组件在同一状态下只使用一个阴影层级 |
| 层级对应海拔 | 组件视觉海拔越高，使用的阴影层级数值越小 |
| 颜色统一 | 所有阴影颜色统一使用黑色半透明，不使用彩色阴影 |
| 禁止随意调整 | 不可在规范值之外自定义阴影参数 |
| 深色模式 | 深色模式下阴影透明度需适当降低或替换为描边方案 |

# 六、Button Design Skill — 按钮系统

## 概述

按钮系统定义了组件库中所有可交互按钮的类型、尺寸、状态与使用规范。按钮是页面中最核心的操作触发元素，通过类型与尺寸的合理搭配，引导用户完成主要操作流程，并通过状态变化提供清晰的交互反馈。

---

## 6.1 按钮类型 Button Types

### 主要按钮 Base Button（Primary）
视觉权重最高，使用品牌主色填充背景。用于页面中最重要的唯一主操作，如提交、确认、保存。**每个视图区域内主要按钮不超过 1 个。​**

### 次要按钮 Secondary Button
使用描边样式，无填充背景。用于与主要按钮并列的次级操作，如取消、返回、重置。

### 辅助按钮 Auxiliary Button
视觉权重低于次要按钮，通常使用浅色背景或更弱的描边。用于辅助性操作或非核心功能入口。

### 虚线按钮 Dashed Button
使用虚线描边样式，常用于新增、上传等引导性操作入口，视觉上具有"待填充"的暗示语义。

### 文字按钮 Text Button
无背景、无描边，仅以文字形式呈现。用于低优先级操作、内联操作或空间极为紧凑的场景。

### 图标按钮 Icon Button
仅包含图标，无文字标签。用于工具栏、操作列等空间有限且图标语义明确的场景，需配合 Tooltip 提示说明操作含义。

---

## 6.2 按钮尺寸 Button Sizes

| 尺寸 | 高度 | 适用场景 |
|------|------|---------|
| Small（小号） | 24px | 表格行内操作、标签旁操作、空间紧凑场景 |
| Medium（中号） | 28px | 表单内操作、卡片内次级操作 |
| Default（默认）
